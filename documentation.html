  4. NOTEWORTHY:
    • Code:
      • Made liberal use of helper functions and data files to reduce repeated code.
      • Tried structuring my game as I would an actual game engine, with the game loop tracking deltaTime, and having an abstracted game loop that doesn't handle game logic.
      • Attempted to give classes a singular purpose to keep a clean object oriented design, and succeeded generally (only class that got excessive was Scene in my opinion)
      • Created clean systems for loading spritesheets into animations, and loading sprites, that allowed me to make additions easily and cleanly
      • Folded my own simple input system that keeps track of input using events so that input checking will always be accurate, no matter the framerate
      • Separated large animation data from code into a data file to clean up the code
      • Used events and promises to ensure that loading assets came before any code dependent on said assets were run
    • Gameplay/User Experience:
      • Balanced the game such that the player has to take an active role in the game (enemy deaths award less than enough for one troop, so the player has to shoot energy balls)
      • Added different troop types that handle differently to add interest to gameplay
      • Added animated assets to make the game feel more alive--troops are in motion whether taking action or just standing around
      • Prevented highlighting on canvas element so clicking wouldn't cause frsutration
